## 二、文件格式与分页规则

### 2.1 图片命名规则

**格式**：`ep{章节编号}_p{页码编号}.jpg`

示例：
- `ep001_p001.jpg` — 第 1 章第 1 页
- `ep001_p015.jpg` — 第 1 章第 15 页
- `ep095_p042.jpg` — 第 95 章第 42 页

**特点**：
1. 章节用 3 位零填充（001、002、...、095）
2. 页码用 3 位零填充（001、002、...、999）
3. 全部为 JPEG 格式，尺寸、质量因来源而异
4. 文件大小通常在 50KB-400KB 之间

### 2.2 CSV 分页索引（关键文件）

`data/pages_index.csv` 的列结构：

```
episode_order, page_order, page_id, image_url, local_file
```

示例记录：

```csv
1,1,5ccb04083478850224b4da8d,https://storage1.picacomic.com/static/xxx.jpg,images/ep001_p001.jpg
1,2,5ccb04083478850224b4da8e,https://storage1.picacomic.com/static/xxx.jpg,images/ep001_p002.jpg
1,3,5ccb04083478850224b4da8f,https://storage1.picacomic.com/static/xxx.jpg,images/ep001_p003.jpg
95,60,5ccb04083478850224b4db23,https://storage1.picacomic.com/static/xxx.jpg,images/ep095_p060.jpg
```

**关键信息**：
1. `episode_order`：章节编号（1, 2, 3, ..., 95）
2. `page_order`：章节内页码（1, 2, 3, ..., n）
3. `page_id`：来自 pica 平台的唯一页面 ID
4. `image_url`：远程图片地址（不可用，图片已本地化）
5. `local_file`：本地相对路径，格式固定为 `images/ep{ep}_p{page}.jpg`

**用途**：CSV 的 `episode_order` + `page_order` 就是每本漫画的**权威阅读顺序**。读取图片时应该按 CSV 记录顺序，而不是按文件名字母序。

### 2.3 JSON 元数据结构

`data/comic_detail.json` 包含的核心字段（以蚀むハイヴ为例）：

```json
{
  "_id": "693c3095aaa57d2d465f22ec",
  "title": "蝕むハイヴ",
  "description": "全彩本",
  "author": "五臓ろっぽ",
  "chineseTeam": "",
  "categories": ["全彩", "短篇", "生肉", "扶他樂園"],
  "tags": ["全彩", "巨乳", "緊身衣", "觸手", "拘束", "扶她", "後入", "中出", "二穴", "馬尾", "惡墮"],
  "pagesCount": 34,
  "epsCount": 1,
  "finished": false,
  "totalLikes": 79,
  "totalViews": 10656,
  "totalComments": 4
}
```

**可利用字段**：
| 字段 | 用途 |
| --- | --- |
| `title` | 漫画标题，用于生成动画提示词时引用作品名 |
| `description` | 简介，提供故事背景线索 |
| `author` | 作者，生成提示词时可标注风格来源 |
| `categories` | 分类，辅助判断漫画类型与目标动画风格 |
| `tags` | 标签，提供画面元素和内容特征 |
| `pagesCount` | 总页数，验证图片完整性 |
| `epsCount` | 总章节数，预分配章节级缓存 |

通过对多本漫画的实际查看，页面特征总结如下：

**关键判断**：

3. 标准日漫格式是固定宽高比的多分镜页，每一页就是阅读序列中的一个独立页。

## 三、五大新需求分析

### 需求 1：样本查看与格式摸底

**状态：已完成。** 已对全部 7 部有效漫画做了目录结构、文件格式、分页规则、CSV/JSON 元数据、页面实物特征的摸底。关键发现均归纳于本文档第一、二章。

### 需求 2：漫画故事连续性理解

**分析**：

故事连续性理解的关键瓶颈是：
1. 页面之间是序列化的（有了 CSV 顺序），但每页内部的多个分镜之间存在隐式的阅读顺序和叙事逻辑。
2. 日漫、韩漫、中文漫的阅读方向不同（右到左、左到右、上到下），这是叙事理解的前提。
3. 对话气泡的指向关系和人物视线方向是连接前后分镜的线索。

**建议方案**：
1. **页面级连接**：直接使用 CSV 的 `episode_order + page_order` 作为页面间顺序，无需额外推理。
2. **页内分镜级连接**：先做分镜切割（需求 3），再通过以下手段理解分镜间叙事关系：
   - 对话文本的承接关系
   - 人物表情和动作的延续性
   - 场景切换标记
   - 留白/黑页作为章节分隔符
3. **跨页连续性**：通过上一页最后一格与下一页第一格的角色、场景、对白对比来判断。

### 需求 3：漫画智能切分 + OCR 文字识别

**分析**：这一需求包含两个子任务：分镜切分和对白文字提取。

#### 3.1 分镜切分

**现状**：参考资源项目 `manga-pipeline-reference` 中已有 comfyui_panels、MangaPanelizer 等现成资源。

**不同漫画类型的切分策略**：

| 漫画类型 | 切分策略 | 推荐工具 |
| --- | --- | --- |
| 标准日漫页（多分镜） | panel detection → polygon → crop | comfyui_panels |
| 纵向条漫（韩漫/国漫） | 纵向滑动窗口扫描空白间距 → 按间距分段 | 自定义脚本 + comfyui_panels 辅助 |
| 全彩写真/画集 | 不切分，整页作为单格处理 | 直接跳过 |

**关键发现**：
- 美丽新世界和继母的朋友们是纵向条漫，传统日漫的 panel detection 方法可能不适用，需要纵向扫描切片。
- 蚀むハイヴ、社團學姊是标准页漫，可直接使用 comfyui_panels 的 Detect Panels In Image 节点。

#### 3.2 OCR 文字识别

**现状**：参考资源项目中已有 Comfyui-ocr、ComfyUI-PaddleOCR、ComfyUI-DeepSeek-OCR。

**建议策略**：
1. **优先使用 Florence-2 或 QwenVL 做端到端图文理解**，而不仅仅是 OCR。
   - 漫画文字通常以气泡、拟声词、旁白的混合形式出现。
   - VLM 可以同时识别文字内容和它在画面中的语义角色。
2. **纯 OCR 作为补充**，用于精确文字提取：
   - PaddleOCR 对中文、日文支持较好
   - 对蚀むハイヴ这样的日文生肉，需要明确的日文 OCR
3. **输出格式**：每格分镜的 OCR 结果应当标记文本类型：

```json
{
  "panel_id": "ep001_p003_panel_2",
  "texts": [
    {"content": "你终于来了", "type": "dialogue", "speaker_guess": "角色A"},
    {"content": "砰——", "type": "sfx", "position": "top-right"},
    {"content": "...", "type": "narration", "position": "top"}
  ]
}
```

### 需求 4：生成漫画转动画提示词

**分析**：这是本次需求中最依赖上下文完整性的环节。一份好的漫画转动画提示词必须同时涵盖画面内容和叙事逻辑。

**推荐提示词结构**：

```
[SCENE]: 场景描述（地点、时间、氛围）
[CHARACTERS]: 角色列表（外貌、服装、当前表情、当前动作）
[ACTION]: 当前发生的关键动作
[CAMERA]: 镜头建议（景别、角度、运动方式）
[DIALOGUE]: 对话 / 旁白 / 拟声词内容
[TRANSITION]: 与上一格/下一格的衔接方式
[STYLE]: 动画风格参考（基于原漫画标签）
```

**生成流程建议**：

1. **单格级**：基于 Florence-2 首轮描述 + OCR 文本 → 生成单格提示词
2. **场景级**：基于 QwenVL 对同一场景内多格分镜的上下文理解 → 补充 TRANSITION 和 CAMERA
3. **角色一致性标注**：将角色注册表中的 `character_id` 注入提示词，确保连续画面中同一角色的外观描述一致
4. **风格参考**：从 `comic_detail.json` 的 `tags` 和 `categories` 字段提取风格词

**示例输出**：

```
[SCENE] 教室门口，黄昏时分，夕阳斜照
[CHARACTERS]
  - 角色A(char_001): 短发黑发女生，校服，表情紧张，站在门口
  - 角色B(char_002): 高个子男生，便服，在背景中举手
[ACTION] 角色A转头看向门口方向
[CAMERA] 中景，从角色A背后略偏侧的角度，轻微推镜向门口
[DIALOGUE] 角色A: "你终于来了"
[TRANSITION] 承接上一格角色A听到门响的反应，引出下一格角色B开口回应
[STYLE] 日系动画风格，柔光，校园题材
```

## 六、风险点

1. **条漫切分**：美丽新世界、继母的朋友们等条漫的纵向切片逻辑与标准页漫完全不同，comfyui_panels 可能不够用，需要补充自定义切片脚本。
2. **日文 OCR**：蚀むハイヴ是日文生肉，PaddleOCR 的日文模型精度需要实测后才能确认。
3. **对话气泡识别**：漫画气泡形状多样（尖角、圆形、云形、无框），VLM 对不规则气泡的文字提取准确性存在不确定性。
4. **跨页连续性**：长篇漫画（如美丽新世界 95 章）的章节间角色和场景延续需要更大上下文窗口。
5. **动画提示词可用性**：生成的提示词实际能否驱动 ComfyUI 或 WanAnimate 出满意的动画效果，需要端到端验证。
