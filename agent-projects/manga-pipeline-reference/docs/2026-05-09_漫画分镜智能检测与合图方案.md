# 漫画分镜智能检测与合图方案

## 问题本质

当前 picaweb 下载的条漫（美丽新世界、继母的朋友们等）在服务端按约 4200px 固定高度纵向切块下发。切割点不关心画面内容——一条完整分镜如果恰好跨过 4200px 边界，就会被一刀斩成上下两半。

这不是"分镜切得不准"的问题，而是**切分逻辑根本没有"分镜"这个概念**——它只认识像素高度，不认识画面语义。

## 核心思路转变

| 旧思路 | 新思路 |
| --- | --- |
| 靠高度阈值（<1200px 就是残片）| 靠内容识别（分镜框线、画面语义断点） |
| 靠像素边界对比（上下图接缝处像不像）| 靠分镜构图逻辑（哪里是"一格的结束"） |
| 先切后补 | 先理解后切 |

正确的做法是：**让算法先看懂整张图上有哪些分镜，再决定切割点应该落在哪里**。如果一个分镜的边界落在两张图的接缝中间，那就应该把两张图拼回一张，而不是硬拆。

## 已检索到的成熟方案

### 1. DeepPanel（最推荐）

- **仓库**：`pedrovgs/DeepPanel`（GitHub，161 stars，Apache-2.0）
- **核心能力**：用深度学习模型识别漫画页中每个分镜的位置
- **方式**：不是 OpenCV 规则，而是训练了一个专门的 CNN 模型做语义分割
- **支持**：Python 项目，有 Android/iOS 库
- **关键描述**：README 第一句就是 "Finding a panel inside a comic page is the hardest thing I've ever done in computer science!"——作者自己承认这是计算机视觉里很难的问题
- **限制**：需要自备训练数据集（仓库不提供训练好的权重），模型主要针对矩形边框分镜，对不规则/重叠分镜能力有限

**对本项目的价值**：提供了"内容感知分镜检测"的完整思路和代码框架。可以直接基于它的模型架构，用本地的漫画数据做微调。

### 2. YOLOv12x Comic Panel Detection

- **资源**：`mosesb/best-comic-panel-detection`（Hugging Face）
- **核心能力**：基于 YOLOv12x 目标检测模型，微调后用于漫画分镜检测
- **方式**：把"分镜"当作目标检测对象，输出每个分镜的 bounding box
- **优点**：模型权重可直接下载，即开即用
- **适合**：标准漫画页面中有明显边框的分镜

**对本项目的价值**：可立即下载试用，对页漫类型（蚀むハイヴ、社團學姊）的分镜检测效果值得优先验证。

### 3. Manga Panel Extractor

- **资源**：`adenzu/Manga-Panel-Extractor`（GitHub）
- **核心能力**：计算机视觉应用，自动检测和提取漫画页中的独立分镜
- **方式**：基于传统 CV 方法（边缘检测 + 联通组件分析）或深度学习

**对本项目的价值**：作为传统 CV 路线和 DL 路线的对比参考。

### 4. Manga109 + CVPR 2025 分割标注

- **资源**：CVPR 2025 论文 "Advancing Manga Analysis: Comprehensive Segmentation Annotations for the Manga109 Dataset"
- **数据集**：`MS92/MangaSegmentation`（Hugging Face）
- **核心能力**：为 Manga109 数据集（109 卷漫画）提供了像素级精度的分割标注，包括分镜框（frame）、文字/对话框（text/dialog）、拟声词（onomatopoeia）等六类对象
- **特点**：标注精度远超简单 bbox，可以做像素级语义分割

**对本项目的价值**：
1. 可以直接用这个数据集训练或微调分镜检测模型
2. 标注的六类对象中，"frame"就是分镜框——正好对应"判断分镜边界"的需求
3. 如果把分镜框检测出来，就能知道画面中哪里是"一格结束、另一格开始"的位置

### 5. Max Halford 的分镜分割博客

- **资源**：`maxhalford.github.io/blog/comic-book-panel-segmentation/`
- **核心能力**：用经典计算机视觉方法（形态学操作 + 联通组件）做漫画分镜分割
- **方式**：不需要深度学习，纯 CV pipeline

**对本项目的价值**：适合快速验证，不需要 GPU、不需要训练。先跑通这条线，确认问题边界，再决定要不要上深度学习。

### 6. Roboflow Manga Panels Detector

- **资源**：Roboflow Universe，1380 张开源分镜图片 + 预训练模型
- **核心能力**：可直接调用的目标检测 API/模型

**对本项目的价值**：额外的标注数据和预训练权重来源。

### 7. 其他学术参考

| 资源 | 说明 |
| --- | --- |
| `Thessal/manga_panel_segmentation` | 使用 U-Net 做漫画分镜分割 |
| `MangaDiffusion` (arXiv) | 分镜感知的漫画生成，包含了 panel splitting 模块 |

## 推荐实现路线

### 第一步：先补"分镜框检测"这个能力

不用急着把切碎的图片拼回去。先回答一个更基础的问题：**这张图上有几个分镜？每个分镜的边界在哪里？**

工具选择：

1. **轻量验证**：先跑 Max Halford 的经典 CV 方案（不需要 GPU，半小时能出结果），看它在本地漫画上的表现
2. **精度提升**：下载 `mosesb/best-comic-panel-detection` 的 YOLOv12x 模型，对比效果
3. **最终路线**：如果需要更高的精度，用 Manga109 的分割标注数据集微调 DeepPanel 或 U-Net 模型

### 第二步：用分镜框信息指导"拼图"

有了每张图上的分镜框检测结果之后，拼图逻辑就清晰了：

```
对每对相邻的图片（图N, 图N+1）：

1. 分别检测图N和图N+1上的所有分镜框
2. 检查图N底部是否存在"不完整的分镜框"（底部无闭合框线）
3. 检查图N+1顶部是否存在"不完整的分镜框"（顶部无闭合框线）
4. 若两者同时存在 + 分镜框的宽度一致 → 判定为同一分镜被拆分 → 合并
5. 否则 → 图像边界恰好也是分镜的自然边界 → 不合并
```

这比像素级对比可靠得多，因为判断依据是"分镜框是否完整"，而不是"像素是不是连续"。

### 第三步：对条漫做纵向智能切片

如果日后有"把一整条连续条漫切成分镜"的需求，思路一样：

1. 把条漫整条加载
2. 用分镜框检测找出所有分镜边界
3. 按边界切割
4. 如果检测不到框（无框条漫），回退到"空白间距检测 + VLM 判断"
