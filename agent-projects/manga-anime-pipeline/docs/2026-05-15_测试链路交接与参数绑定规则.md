# 漫画短视频测试链路交接与参数绑定规则

## 核心想法

这个项目现在处在“先做可控小实验，再把可靠部分收进正式工作流”的阶段。不要急着把漫画一次性变成完整短片；先把章节理解、角色设计、风格矩阵、图生视频模型横评拆开跑，每一步都输出能人工审核的文件。

新规则是：任何 ComfyUI 测试输出都必须和生成它的参数绑定。图片、视频旁边要有同名 `*.provenance.json`，里面保存完整 API workflow、节点输入、模型、LoRA、VAE、采样器、seed、尺寸、提示词、prompt_id 和任务上下文。这样以后看到一个成品，就能直接知道它怎么来的，也能拿参数继续调。

## 参数绑定规则

每个 ComfyUI 输出文件必须有一个同名旁车：

```text
char_dark_ponytail_01.png
char_dark_ponytail_01.png.provenance.json
```

旁车文件必须至少包含：

- `workflow_api`：完整 ComfyUI API workflow。
- `extracted_parameters.models`：checkpoint、UNet、LoRA、text encoder、VAE 等模型节点。
- `extracted_parameters.samplers`：seed、steps、cfg、sampler、scheduler、denoise、分段采样参数。
- `extracted_parameters.vae`：VAE loader / decode / encode 相关节点。
- `extracted_parameters.dimensions`：width、height、length、fps、batch_size。
- `extracted_parameters.prompts`：正向、负向和其他文本编码节点。
- `prompt_id`、`client_id`、`extra_data`、`task_context`。

统一工具在：

```powershell
pipeline\comfy\provenance.py
```

新脚本只要复制或生成了图片、视频，就调用：

```python
from pipeline.comfy.provenance import write_output_provenance_files

sidecars = write_output_provenance_files(
    copied_outputs,
    project_root=PROJECT_ROOT,
    workflow=workflow,
    workflow_name="your_workflow_name",
    prompt_id=prompt_id,
    client_id=client_id,
    extra_data=extra_data,
    task_context=task_context,
)
```

如果只是 Stage 6 记录了 ComfyUI 原始输出文件名、没有复制文件到项目 runtime，`pipeline/comfy/submitter.py` 会在 `runtime/.../comfy/provenance/` 生成镜像 sidecar。后续更好的做法是把成品也复制回项目 runtime，再使用同名旁车。

## 给新对话的接手建议

先看三个入口：

1. 读 `README.md` 的“角色圣经与本地模型建议”“角色风格矩阵测试”“角色图生视频模型横评”。
2. 读本文档，确认本轮测试结论和参数绑定规则。
3. 看 `runtime/.../review/*/review_index.md` 和 contact sheet，先做人眼判断，再改脚本参数。

调参顺序建议：

1. 先改提示词 profile，不急着换模型。
2. 再比较 checkpoint / UNet / LoRA。
3. 角色仍漂移时，引入 IPAdapter、ControlNet、角色 LoRA 或参考图编辑链。
4. 视频只在 25 帧短样稳定后再加长；如果短样已经崩，长视频只会更难审核。
