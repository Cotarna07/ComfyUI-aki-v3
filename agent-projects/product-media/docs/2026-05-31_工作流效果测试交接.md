## VLM 提示词功能

批量测试不同商品时，VLM 生成专属提示词是必要功能。原因是不同商品的结构、配件、人物位置、颜色和材质约束不同，通用 prompt 容易漏掉关键 SKU 事实。

现有脚本支持：

```powershell

  --plan .\agent-projects\product-media\runtime\product_image\workflow_effect_tests_20260531\test_plan.json `
  --stage all `
  --use-vlm
```

默认接入 `agent-projects/product-vlm-review` 的 Ollama Qwen3-VL 后端，逐图识别并按产品缓存：

```text

```

同一产品的 prompt plan 会被多个工作流、多个 seed 复用。需要强制重新识别时使用 `--refresh-vlm`。

注意：VLM 只用于批量提取产品事实、生成提示词和辅助风险初筛，不能单独放行真实商品图。最终仍需按原图人工或二次 VLM 对照验收。

- 运动模糊只作用于背景；商品贴回后始终清晰。
- 单独跑：`python scene_compose.py <scene_jobs.json>`。

## 2026-05-31 续三：场景图调优与二次验收

在场景生成基础上补齐四项能力，均已接入 `scene_compose.py` / `scene_verify.py` 与一键脚本。

### 1) 多 seed A/B
`scene.seeds: [a, b]` 一次出多版，挑最好。已验证警车 seedA/seedB 出不同街景构图。

### 2) 替代主题提示词
直接改 `scene.prompt` 即可换主题：已验证警车"白天棕榈大道追逐"、苦力怕"夜间熔岩洞穴"。

### 3) 构图控制（`scene.layout`）
把商品紧致抠出后重新摆到指定画布：`{aspect, product_scale, anchor, margin}`。
- `aspect`：`4:5` / `16:9` / `1:1` 等；`anchor`：`bottom-center` / `left` / `center` …；`product_scale`：商品高度占画布比例。
- 已验证 SF-24 竖版 4:5（上方留标题位）、警车 16:9 横幅（商品靠左、右侧留文案位）。
- 原理：商品摆到中性灰画布 → mask 取反 → SDXL 重绘整块背景 → 贴回原商品。

### 4) 真实性二次验收（`scene_verify.py` / `--stage verify`）
用 qwen3-vl 对照"原始商品图 vs 生成场景图"，只判商品本体一致性（背景不同不算失败），输出 PASS/FAIL + 理由到 `<batch>/verify/`。
- 一键：`--stage verify`（会先 `/free` 让显存给 Ollama）；可 `--skip-verify`。
