3. 这个工作流怎么用

在 ComfyUI 网页里运行

1. 打开 ComfyUI 网页: http://127.0.0.1:8188
2. 载入工作流 JSON:
   agent-skills/comfyui/workflows/api/wan22_i2v_dasiwa_six_loras_4080_safe.json
3. 把 LoadImage 节点的起始图换成你自己的图
4. 改正向提示词和负向提示词
5. 先不要放大分辨率，也不要先把 length 改长
6. 点击 Queue Prompt 运行

4. 为什么你在 Chrome 里看不清参数

因为你直接打开的是 API JSON，它本来就是给 ComfyUI 后端提交用的，不是给人阅读的 UI 文件。Chrome 能显示文本，但不会帮你把节点关系、模型分支、LoRA 分组和关键参数整理成可读面板。

这个 HTML 页就是为了解决这个问题。以后你想先在浏览器里审参数，直接打开这个文件，不要先看原始 JSON。
如果你想看节点连线和完整画布，还是得把 JSON 导入 ComfyUI 网页。浏览器直接打开原始 JSON，只适合机器，不适合人看。

5. 我可以通过 API 查看到吗

可以。现在这台机的 ComfyUI API 是通的，我已经确认过 /object_info 和 /history 都能访问。你可以用 API 做三件事：

查看节点和可用模型: /object_info 提交任务: /prompt 查看历史与输出: /history

查看历史任务

GET http://127.0.0.1:8188/history

查看节点注册情况

GET http://127.0.0.1:8188/object_info

提交这个工作流

POST http://127.0.0.1:8188/prompt
Content-Type: application/json

{
  "prompt": { ...把工作流 JSON 整体放这里... },
  "client_id": "chrome-preview"
}

6. PowerShell API 示例

$path = 'D:/ComfyUI-aki-v3/agent-skills/comfyui/workflows/api/wan22_i2v_dasiwa_six_loras_4080_safe.json'
$workflow = Get-Content -Raw -LiteralPath $path | ConvertFrom-Json
$body = @{
  prompt = $workflow
  client_id = 'wan22-dasiwa-preview'
} | ConvertTo-Json -Depth 100

Invoke-RestMethod -Uri 'http://127.0.0.1:8188/prompt' `
  -Method Post `
  -ContentType 'application/json' `
  -Body $body
提交后记住返回的 prompt_id，再去 /history 里查这个任务的运行状态和输出文件。
