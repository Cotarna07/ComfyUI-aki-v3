# 商品图优化与真实性守门

## 目标

在提升场景、灯光与广告表现力的同时，阻止生成结果被误用为不真实的商品展示图。将“美术质量”与“商品忠实度”分别验收；漂亮但与实物不符的图不能进入真实展示交付。

用户所说的“产品图过拟合”在没有训练微调的编辑流程中通常表现为商品身份漂移或结构幻觉：模型迎合广告风格而重画了实物。无论术语如何，只要输出与可验证实物不一致，都按发布阻断风险处理。

## 必读与边界

1. 先读取工作区 `AGENTS.md`、`agent-skills/README.md`、`agent-skills/comfyui/SKILL.md`、`agent-skills/comfyui/registry.json` 和 `agent-skills/docs/comfyui_api_rules.md`。
2. 涉及本轮积木商品案例或需要复用已测参数时，再读取 `agent-skills/docs/2026-05-26_产品图展示视频排障优化报告.md`。
3. 执行前查询实际 ComfyUI `/system_stats`、`/object_info`、可用模型与队列状态；不得凭文档猜设备、节点或模型。
4. 仅在用户明确允许时写入 `ComfyUI/` 原区；API 输出与用户授权的模型安装除外。

## 先声明交付类型

生成前选择一种轨道，并在 `extra_data.notes` 和回复中标明：

| 轨道 | 允许的变化 | 禁止事项 | 对外用途 |
|------|------------|----------|----------|
| `factual_product` 真实商品展示 | 背景、地面、灯光、非商品区域清理、轻微镜头裁切 | 改结构、颜色、材质、配件数量、人物数量、贴纸/印花、比例、摆位关系 | 商品页主图、功能/规格展示、成交素材 |
| `creative_campaign` 创意广告主视觉 | 强化背景、光影、叙事、合理构图重排；仅在明确标注下允许主体演绎 | 冒充实物照片或宣称配件/结构完全一致 | 广告吸引图、故事海报、视频开场 |
| `diagnostic_preview` 诊断预览 | 为测试流程使用更快配置或有限样张 | 当作最终效果或对外发布 | A/B、提示词和模型选型 |

当用户没有明确说明且素材面向商品销售时，默认采用 `factual_product`；需要更强创意时，另出 `creative_campaign`，不得覆盖真实展示结果。

## 执行流程

### 1. 建立商品身份清单

在生成前从原始图片提取必须不变的事实，至少记录：

- 商品类别、主色、材质观感与可辨识轮廓。
- 核心组件及相对位置，例如塔楼、侧棚、车翼、驾驶舱、轮胎数、武器、发动机舱。
- 人仔、动物、配件的数量、颜色与关键外观。
- 印花、徽标、贴纸、透明件和特殊零件等强识别特征。
- 包装、尺寸标注和宣传文字：它们是验证来源，不应被生成图虚构。

使用本技能目录下 `references/fidelity-gate.md` 的清单模板。若只有包装图而没有足以验证实物结构的视角，先说明证据不足，不把重绘结果标为真实展示。

### 2. 选择输入与预处理

- 优先使用无文案、主体完整、背景干净的产品图作为编辑输入。
- 对包装、标题、箭头或尺寸标注干扰主体的图片，先使用裁切或遮罩去除非商品区域，再编辑背景。
- 不要依靠一句“移除文字”让模型同时重建被文字覆盖或邻近的商品结构。
- 对深色商品，优先选择浅色平台、柔和晨昏光或明亮棚拍式场景；深色夜景常使结构不可验收。

### 3. 选择本机测试路线

- 先在当前设备上选择能稳定执行且效果尽可能好的配置，生成代表性本机结果；记录主模型、精度、VAE、编码器、LoRA、输入、seed、采样、guidance、预处理、耗时和输出。
- 同一商品至少保留一张保守的 `factual_product` 候选，再按需要生成 `creative_campaign` 候选。
- 仅在本机确认某路线能提供实用质量后，才规划云端运行同系更大或更高精度配置。
- 不兼容的模型/LoRA 不混挂。例如 SDXL LoRA 不直接加载到 Flux Kontext 或 Qwen Image Edit。

### 4. 编写受控提示词

按以下顺序写提示词：

1. **商品不变量**：列出必须保留的轮廓、组件、颜色、数量与相对关系。
2. **允许编辑项**：仅描述背景、灯光、地面、镜头、空气感等允许变化。
3. **可读性要求**：商品占画面比例、明暗分离、接触阴影、关键细节清晰。
4. **禁止变化项**：不增删配件、不改材质/构造/数量、不生成文字水印包装。

在 `factual_product` 中避免使用 `reimagine`、`action tableau`、`allow repositioning`、`transform into` 等鼓励主体重画的指令。创意轨道使用这些表达时必须清楚标注为创意图。

### 5. 提交与留痕

- 通过可复现 API 工作流执行，`client_id` 使用 `agent:<agent>|workflow:<workflow>|run:<id>`。
- 在 `extra_data.notes` 记录轨道、商品标识、输入来源、预处理、模型与全部覆盖参数、提示词、预期验收重点。
- 保存原图、输入副本、输出与必要的对照记录；禁止只留下“最好看”的输出而丢失验证来源。

### 6. 强制真实性验收

在接受输出前，逐张与原始素材对照，使用本技能目录下 `references/fidelity-gate.md`。

- 对 `factual_product`：任一关键结构、配件、数量、比例、颜色、透明件、印花或角色身份变化即判失败；不得通过“整体很像”放行。
- 对 `creative_campaign`：允许已声明的叙事变化，但必须列出与真实商品不一致之处，并明确标注“创意广告图，不用于核验实物内容”。
- 无法确认的部位按风险处理：不得宣称一致，补取其他原图视角或降级为创意结果。
- 对“锁前景 + 生成背景”的真实展示路线：必须在叠加商品之前审查独立背景底板是否含有同类商品、配件、包装、伪文字或 logo；验收方式见 `references/visual-gate-workflow.md`。
- VLM 只提供风险证据和重试建议，不能单独放行真实商品结果。布尔结论与证据矛盾时，按证据中的风险拒绝。

## 已验证经验

- 白底完整套装适合整图编辑以统一背景、接触阴影和环境光，结构保留通常最好。
- 文字/包装干扰的载具图先裁切或遮罩，再做场景化，较不容易出现缺件与误改。
- 深色载具套用暗夜电影感容易压黑产品；强行提亮整图又可能将积木重绘成真实车辆。使用浅色环境并强调锁定结构更稳。
- 让模型产生戏剧性动作、人物关系或低机位海报构图会明显增加 SKU 漂移风险，只能优先进入创意轨道。
- 视频阶段优先对通过验收的静帧制作可控镜头运动和后期光效；生成式 I2V 会重新改写主体，应逐帧验收。

更多本轮输出表现、测试参数与可复用提示词模式见本技能目录下 `references/tested-patterns.md`。

## 失败处理

- 发现 SKU 漂移：不要继续微调同张结果冒险放行；回到更干净输入、收紧不变量、降低主体重排幅度，或改用裁切/遮罩/局部编辑路径。
- 发现画面高级但事实错误：保留为标注清楚的创意候选，不得进入商品真实展示分支。
- 工作流反复在美术质量与保真间冲突：向用户报告模型边界，保留真实基线，并提出下一项本机 A/B 或已验证路线的云端放大方案。
- 背景生成出现在商品合成前即可识别的禁物时：拒绝底板并更换 seed、收紧背景提示词或回退空棚，不允许以最终合成图“看起来不明显”为理由通过。
