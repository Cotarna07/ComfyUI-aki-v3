# 漫画转 AI 动漫与舞蹈动感方案

生成时间：2026-04-21

## 结论

当前 4080 本机可以做原型验证，但不适合作为最终高动感视频生产的唯一算力。现有僵硬感主要来自四点：

1. 使用的是兼容性链路：`WanAnimateToVideo` 被普通 Wan2.2 I2V 模型驱动，而不是专用 WanAnimate / VACE / LTX Pose 模型。
2. 动作参考视频是近景、低机位，DWPose 抽出的腿部和躯干信息不足，只有上半身和手势比较稳定。
3. 没有做节拍驱动的关键帧规划，只是把原 BGM 合回去；动作没有在 beat 上人为强化。
4. 没有经过补帧、运动模糊、镜头推拉、二次元线稿修复等后处理，16fps 会显得更僵。

要做“有感觉”的跳舞视频，建议使用多工作流流水线，而不是单一工作流一次生成到底。

## 本地可用基础

- 已可用：
  - `Wan2.2 T2V/I2V` high/low noise + LightX2V LoRA。
  - `WanAnimateToVideo` 节点可运行。
  - `DWPreprocessor` 可提取身体、手、脸姿态。
  - `RIFE VFI` / `FILM VFI` 节点已加载。
  - `video_upscale_gan_api` + `RealESRGAN_x4plus.safetensors` 可用。
  - `animagineXLV31_v31.safetensors` 已通过硬链接接入主 checkpoints。

- 当前缺失，且会直接影响质量：
  - `Wan2_2-Animate-14B_fp8_e4m3fn_scaled_KJ.safetensors`
  - `WanAnimate_relight_lora_fp16.safetensors`
  - VACE 14B / 1.3B 视频编辑模型
  - LTX-2 19B/20B 与 pose/depth/canny/control LoRA
  - FlashVSR 或同类视频超分模型

## 候选工作流分层

### 1. 角色与漫画关键帧

目标：先保证角色是二次元、身份稳定，再谈动起来。

- `sdxl_animagine_anime_dancer_reference.json`：当前已新建，可生成二次元角色参考图。
- `FLUX工作流合集/AI固定角色+精准大动态姿势生成技巧/工作流/动漫文生图工作流.json`
- `FLUX工作流合集/FLUX解决人物一致性工作流/生成人物形象.json`
- `电商.../番茄酱-动漫真人互转/真人和卡通转换/真人转漫画转真人.json`
- `电商.../（人物动作）人物一致性 姿势随心变/一张图生成多动作(保持服装一致性).json`

建议：漫画项目优先建立“角色圣经”：正面、侧面、背面、表情表、服装表、色卡。后续每个镜头从这个角色圣经出关键帧。

### 2. 动作控制与舞蹈

目标：把参考动作拆成可控条件。

- `extract_dwpose_from_reference_video.json`：当前已新建，可提取 DWPose 帧序列。
- `4.工作流/workflows/6.视频生视频/03.Wan2.2-Animate-动作迁移.json`
- `4.工作流/workflows/6.视频生视频/05.WanAnimate动作+表情迁移.json`
- `4.工作流/workflows/6.视频生视频/06.WanAnimate 人物A的动作+B的表情混合迁移.json`
- `4.工作流/workflows/6.视频生视频/07.WanAnimate+姿势参考2.0.json`
- `4.工作流/workflows/6.视频生视频/16.好用-Sec-人物迁移_动作迁移_长视频循环.json`
- `MimicMotion 上传图片生成跳舞 图生视频 图片跳舞 (1).json`

建议：舞蹈参考视频必须尽量用全身、无遮挡、节拍清晰的素材。近景胸像参考会导致骨架缺失，生成自然会僵。

### 3. 漫画转动漫视频

目标：从漫画分镜变成连续镜头，而不是把整页漫画直接丢给视频模型。

- 分镜拆分：页面 -> panel -> 镜头清单。
- 每格漫画：去字、补背景、分层角色/背景/道具。
- 关键帧生成：用漫画 panel 作为线稿/构图控制，用角色圣经作为身份控制。
- 动画生成：
  - 静态对白镜头：I2V + 轻微头动/眨眼/镜头推拉。
  - 动作镜头：Pose/Depth/Canny/LTX/WanAnimate。
  - 转场镜头：首尾帧 / VACE / LTX guide。

本地相关工作流：

- `ComfyUI/blueprints/Pose to Video (LTX 2.0).json`
- `ComfyUI/blueprints/Depth to Video (ltx 2.0).json`
- `ComfyUI/blueprints/Canny to Video (LTX 2.0).json`
- `电商.../视频转绘（vace）/Wan2.1_VACE视频转绘工作流.json`
- `电商.../视频转绘工作流/视频转绘稳定工作流.json`
- `电商.../转汇文件/转绘完整版v3-修脸.json`

### 4. 后期增强

目标：让“生成感”变成“动画片感”。

- 补帧：RIFE / FILM，把 16fps 提升到 24/30/48/60fps。
- 高清：RealESRGAN、FlashVSR、视频 detailer。
- 线稿修复：关键帧抽帧修脸/修手，再回灌。
- 节拍：按 BGM beat 做速度重映射、关键帧重排、镜头切点。
- 画面：加轻微运动模糊、镜头推拉、背景视差、色彩统一。

本地相关工作流：

- `video_upscale_gan_api`
- `4.工作流/workflows/8.高清放大类/03.FlashVSR视频高清化工作流-好用.json`
- RIFE/FILM 节点已加载
- `Video Stitch` 可做对比评测

## 推荐测试矩阵

### 测试 A：舞蹈动作

同一个二次元角色，同一段全身舞蹈参考，比较：

1. 当前兼容版 `WanAnimateToVideo + Wan2.2 I2V`
2. 专用 `Wan2.2 Animate` 模型
3. `WanVideoWrapper WanVideoAnimateEmbeds`
4. `LTX Pose/Guide`
5. `MimicMotion`

评估项：动作幅度、节拍命中、角色一致性、脸手稳定、二次元感、运行时间、显存。

### 测试 B：漫画分镜

选择 3 页漫画，拆 8-12 个镜头：

1. 对话近景
2. 转头/眨眼
3. 情绪表情
4. 跑动/打斗
5. 镜头推拉/横移

每个镜头控制输入不同：lineart、depth、pose、首尾帧。

## 云算力建议

- 本机 4080 16GB：适合 512x768、短片段、小样、工作流验证。
- 24GB 显存：适合 720p、部分 14B fp8、短片段正式生成。
- 48GB 显存：推荐主力，适合 WanAnimate / VACE / LTX 19B、较长上下文、较高分辨率。
- 80GB 显存：适合长镜头、高分辨率、LoRA 训练、多模型同时加载。

建议先租 48GB 卡做 1-2 天模型验证，不急着上 80GB。等确定最终路线后，再按批量产出阶段升级。
