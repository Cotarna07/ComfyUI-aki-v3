# 速卖通产品素材优化方案

> **创建时间**：2026-05-19
> **适用平台**：ComfyUI（秋叶 Aki 启动器 v3）
> **GPU 说明**：本文历史版本按 RTX 4080（16GB）整理；2026-05-26 当前 ComfyUI `/system_stats` 实测运行设备为 NVIDIA GeForce RTX 5070 Ti（约 16GB VRAM）。最终质量不受本地设备封顶，算力不足时使用云 GPU。

---

## 一、需求概述

利用本地 ComfyUI 对速卖通产品图片和视频进行优化，主要包括：

| 需求类型 | 具体场景 |
|----------|----------|
| 主图精修 | 去脏点、去反光、局部替换、背景优化 |
| 场景图生成 | 产品放在不同背景/场景中展示 |
| 模特图生成 | AI 生成穿戴/使用产品的模特效果图 |
| 比例适配 | 1:1 主图扩成 3:4 竖版、16:9 横版、详情页长图 |
| 视频优化 | 产品图转轻动效短视频、已有视频超分清晰化 |
| 素材理解 | 从现有主图/视频提取卖点文案、生成英文提示词 |

## 五、推荐实现链路

### 5.1 主图精修链路

```
原始产品图
  ↓
[抠图] RMBG / SAM2 → 白底主图
  ↓
[局部重绘] qwen_image_inpaint_api → 去脏点/去反光/替换局部
  ↓
[扩图] qwen_image_outpaint_api → 适配不同比例（1:1 / 3:4 / 16:9）
  ↓
[超分] UltimateSDUpscale → 高清化输出
```

### 5.2 场景图/模特图链路

```
白底产品图
  ↓
[加载模板] Flux一键电商场景模特生成 / 电商产品摄影工作流
  ↓
[提示词改写] prompt_enhance_api → 生成英文场景描述
  ↓
[图生图/文生图] FLUX + IP-Adapter → 生成场景图/模特图
  ↓
[精修] qwen_image_edit_api → 微调细节
```

### 5.3 产品短视频链路

```
选定产品主图
  ↓
[提示词改写] prompt_enhance_api → 生成运动描述
  ↓
[图生视频] wan22_i2v_api → 生成 3-5 秒轻动效
  ↓
[超分] video_upscale_gan_api → 清晰化
  ↓
[对比评审] video_stitch_api → before/after 拼接
```

### 5.4 现有视频优化链路

```
原始产品视频
  ↓
[分析] video_caption_gemini_api → 提取卖点/生成描述
  ↓
[超分] video_upscale_gan_api → 清晰化
  ↓
[精修] ltx2_v2v_detailer_api → 逐帧细节增强（可选）
  ↓
[对比评审] video_stitch_api → before/after 拼接
```

## 七、注意事项

### 7.1 内容合规

- 产品图必须真实反映商品，不得虚构材质、配件、尺寸或功能
- 模特图需注意肖像权，AI 生成的人物不应用于冒充真人代言
- 平台审核趋严，建议保留原始素材和编辑记录备查

### 7.2 显存管理

- 当前运行设备以 ComfyUI `/system_stats` 实测为准；本地运行 FLUX/SDXL 时可关闭其他占用显存的程序。
- 用户要求效果最好时，最终商品图与成片不得为了适配本机显存而默认降低 `width`、`height`、`length`、模型精度或采用明显较低质量模型。
- 降尺寸、FP8、蒸馏或加速 LoRA 只可用于明确标记的预览、诊断和构图筛选；若优选的最终流程需要更多显存，应记录参数并转用云 GPU 执行。

### 7.3 工作流管理

- 电商专用工作流建议从用户工作流目录复制到 `agent-skills/comfyui/workflows/01-shared/` 再做 API 导出
- 每次修改工作流后重新导出 API JSON，覆盖旧版本
- 保持注册表 `registry.json` 与实际工作流文件同步
