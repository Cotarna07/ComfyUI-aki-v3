# 插件路线图

这份文档说明当前 Aki 包里已经有什么、现阶段真正的短板在哪里，以及下一步最值得继续补哪些插件。

## 当前基础

当前工作区里已经有一批很实用的本地视频工作流插件：

- ComfyUI-LTXVideo
- ComfyUI-WanVideoWrapper
- ComfyUI-VideoHelperSuite
- comfyui_controlnet_aux
- ComfyUI-Impact-Pack
- ComfyUI_IPAdapter_plus
- ComfyUI_UltimateSDUpscale
- rgthree-comfy
- ComfyUI-RMBG
- ComfyUI-Manager

这意味着你已经具备了以下基础能力：

- Wan 和 LTX 两套视频生成体系
- 控制图条件输入
- 掩码和背景处理工具
- 工作流图形操作便利性
- 基本的视频输入输出能力

## 这次已经补上的插件

这次我已经把下面三类高价值插件代码装进了 custom_nodes：

- ComfyUI-KJNodes
- ComfyUI-Frame-Interpolation
- ComfyUI-segment-anything-2

当前状态说明：

- 插件代码已经克隆到本地。
- KJNodes 和 Frame Interpolation 的大部分 Python 依赖已经开始补齐。
- 由于 ComfyUI 正在运行，opencv 相关文件被占用，相关升级需要在重启后继续完成。
- 这些新节点需要在 ComfyUI 重启后才会被实际加载。

## 当前栈的主要短板

虽然基础已经不错，但现在还有几个明显短板：

1. 插帧链路还没有真正进到日常工作流里。
2. 视频修补和图像重绘前的遮罩准备还不够自动化。
3. 分析类 skill 目前主要依赖 Gemini，缺少更成熟的本地视觉语言备选路线。
4. 时间轴式的视频拼接、过渡和批量后处理工具还不够丰富。

## 下一步最值得补强的方向

### 高优先级

#### ComfyUI-KJNodes

- 价值：它是非常实用的视频后处理和工作流胶水节点包。
- 最适合：让 video_stitch、detailer、对比视频、时间控制这类流程更灵活。
- 优先级原因：不引入新的大模型家族，但能明显提升现有视频工作流的可塑性。

#### 插帧插件

- 已安装候选：ComfyUI-Frame-Interpolation
- 价值：Wan 和 LTX 能生成镜头，但插帧才是把预览片段变得更顺滑的下一步。
- 最适合：把 12 到 16 fps 的预览片段做成更平滑的成片或评审片。
- 优先级原因：对观感提升非常直接，而且不要求你重做提示词。

#### SAM2 级别的遮罩插件

- 已安装候选：ComfyUI-segment-anything-2
- 价值：wan_vace_api 和 qwen_image_inpaint_api 的瓶颈常常不是模型本身，而是遮罩做得不够快、不够稳。
- 最适合：主体抠图、物体选区、自动遮罩、视频局部修补前的准备。
- 优先级原因：同时提升图像编辑和视频修补两条线。

### 中优先级

#### 本地视觉语言插件

- 例子：Florence-2 一类的本地图文理解插件
- 价值：现在 image_caption 和 prompt_enhance 已经能直接跑，但依赖 Gemini。后面可以补一条离线分析路线。
- 最适合：本地图片说明、提示词起草、参考图预分析。
- 优先级原因：很有价值，但短期收益不如插帧和遮罩直观。

#### 诊断和性能插件

- 例子：Crystools 一类的运行监控和调试插件
- 价值：长视频和大模型工作流很需要更好的显存、队列、节点耗时可视化。
- 最适合：定位卡慢、查显存波动、追踪执行瓶颈。
- 优先级原因：偏运维体验提升，而不是直接增加创作能力。

### 视目标而定

#### 人脸修复或肖像增强插件

- 价值：只有在你的内容明显偏人像、特写、主播口播时才很值得。
- 最适合：人脸清晰化、面部细节修补。
- 优先级原因：窄场景高价值，不是通用第一优先级。

#### 更完整的 3D 工具链

- 价值：只有在 Hunyuan3D 的实验要继续走向可用 mesh 或资产时才必要。
- 最适合：图像转 3D 后的进一步建模和导出。
- 优先级原因：能力很强，但不属于当前视频学习主线。

## 推荐安装顺序

如果目标是在不把环境搞复杂的前提下持续增强这套项目，建议按这个顺序推进：

1. 先完成 ComfyUI-KJNodes 的启用和验证
2. 再把 ComfyUI-Frame-Interpolation 的依赖补齐并做一条插帧测试链
3. 再把 ComfyUI-segment-anything-2 接进 qwen_image_inpaint 和 wan_vace 的遮罩准备流程
4. 后面再补本地视觉语言插件
5. 最后补诊断类插件

## 现在不该优先做什么

现阶段不建议继续优先加更多 Wan 或 LTX 包装器。你当前这套环境里，视频模型家族已经够学习和对比了。更高回报来自：

- 把后处理做好
- 把遮罩链打通
- 把分析和修复链串起来

## 一个简单判断原则

- 如果某个插件能增强 Wan / LTX 周边的控制、遮罩、后处理，它通常值得优先测试。
- 如果某个插件只是再加一套重叠的文生视频模型，而你当前这套还没吃透，就先放一放。
