# 工作区 Git 与根目录规则

- agent-skills/ 与 agent-projects/ 下由代理协作生成和维护的文件，默认统一由当前工作区现有版本管理跟踪。
- 代理不负责初始化、迁移或调整根目录任何 .git 或 overlay 元数据。
- 秋叶启动器原区文件继续由其自身仓库管理；未经用户明确允许，不要把原区文件并入 agent 自有区的版本管理范围。
- 非用户明确要求，不要在 agent-skills/ 或 agent-projects/ 子目录内再初始化或保留并行的独立 Git 仓库。
- 如果引入了自带 .git 的外部项目，先停用或迁出其子仓库元数据，再决定是否并入当前工作区现有版本管理。
- 现有根目录兼容入口脚本只作为读取或执行入口使用；除用户明确允许外，不作为默认修改目标。
